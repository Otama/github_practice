* Git解説
** 解説
- gitにはブランチ（branch）の概念がある
- 最初にあるのはmasterブランチ
- masterは一番大切なブランチであり，常に正常に動作する状態にする
- 新しい作業を開始するときは必ず新しいbranchを作る
- 後に，作業内容をmasterに取り込む（merge）
